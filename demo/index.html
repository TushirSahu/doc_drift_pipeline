Platform

Everything between your docs and a trusted answer

A serving path with production safeguards, and a quality gate that treats answer quality like a test suite.

Agentic retrieval

The model decides when and how often to search — multi-hop questions get a second retrieval pass instead of a wrong answer.

Layered guardrails

Injection filtering on input, grounding and citation checks on output, and a prompt-leak canary — every answer gets a verdict.

Citation audit

Citations aren't decoration. Each [Source] is verified against what was actually retrieved — and auto-corrected when the model miscites.

Drift gate in CI

Ragas scores every change against a committed baseline. Faithfulness drops below threshold? The build fails before users notice.

Multi-LLM benchmark

Candidate models are scored head-to-head on your own docs; the champion automatically becomes the serving model.

Full observability

Per-request traces, p50/p95 latency, guardrail pass rates, cache hit rates — durable across restarts via Postgres.

Pipeline

From markdown to a gated deploy

Five stages. The last one is the point: quality regressions never reach production silently.

01

Ingest

Markdown is chunked, embedded, and indexed in Qdrant — re-ingested only when content actually changes.

02

Retrieve

Dense search with MMR de-duplication, plus optional BM25 hybrid fusion and reranking.

03

Answer

An agent loop with whitelisted tools writes a grounded answer — every claim cited to its source file.

04

Evaluate

Synthetic QA scored with Ragas: faithfulness, relevancy, precision, recall, correctness.

05

Drift gate

Scores compared to the committed baseline. A regression fails the build — bad answers never ship.
